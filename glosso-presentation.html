THE SOLUTION

One wallet interface.
Three key management systems.

🔀
Core
GlossoWallet

A unified wallet interface that exposes the same four methods — getAddress, getBalance, sign, send — regardless of how keys are stored or managed. Write your agent once, swap the key management system with a single config change. Your agent code never changes.

Sovereign — local encrypted keys
Privy — secure server enclaves
Turnkey — hardware security modules
Same interface, same agent code — different security guarantees

ADAPTER 1

Sovereign Mode

Full self-custody. Keys are generated locally, encrypted with AES-256-GCM, and never leave the agent's environment. No raw private key is ever stored in plaintext.

🔐
AES-256-GCM with PBKDF2 key derivation (100K iterations)
🌱
BIP39 mnemonic seed phrase generation
✍️
Decrypted only at sign-time — wiped from memory immediately
🛡️
Tamper detection via authentication tags

SOVEREIGN

HD Wallet Derivation

A single encrypted seed generates an infinite tree of purpose-specific wallets using BIP44 hierarchical deterministic derivation. Each sub-wallet is isolated — compromising one doesn't expose the others.

✓ Index 0 — Primary wallet (holds main funds)
✓ Index 1 — Trading wallet (DeFi operations)
✓ Index 2 — Vault wallet (long-term storage)
✓ Index N — extensible for any purpose

ADAPTER 2

Privy Adapter

Keys live in Privy's secure server enclaves — never extracted, never exposed. Signing happens remotely via authenticated API calls. Ideal for teams that want managed custody without trusting a single machine.

🏦

🔏
Signing via server-side API — keys never leave the enclave
🔑

ADAPTER 3

Turnkey Adapter

Enterprise-grade key management. Keys live in hardware security modules (HSMs) — the same infrastructure banks and institutions use. Signing requires cryptographically stamped API requests.

🔒
Keys in tamper-proof hardware modules
📜
API key stamping for every signing request
🏢
Organization-scoped wallet management
⚡
Same WalletAdapter interface — zero code changes
